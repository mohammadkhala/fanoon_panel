# تحليل صفحة نقطة البيع (POS) — `/admin/pos`

**الرابط:** `http://127.0.0.1:8000/admin/pos`  
**الهدف:** توثيق المميزات الحالية، الفجوات، والتحسينات المقترحة مع مراعاة سهولة الاستخدام.

---

## 1. المميزات الحالية

### 1.1 قسم المنتجات (يسار الصفحة)
| الميزة | الوصف |
|--------|--------|
| **فلتر التصنيف** | قائمة منسدلة (All Categories + تصنيفات نشطة) — تغيير التصنيف يعيد تحميل الصفحة مع `category_id` |
| **بحث بالكلمة** | حقل بحث مع إعادة تحميل بالـ query `keyword` |
| **بطاقات المنتجات** | عرض المنتجات (صورة، اسم مختصر، سعر بعد الخصم) مع زر "Add to cart" أو عداد الكمية في السلة |
| **ترقيم الصفحات** | Pagination مع الحفاظ على query string |
| **نقرة على البطاقة** | فتح نافذة Quick View للمنتج |

### 1.2 نافذة Quick View (معاينة سريعة)
| الميزة | الوصف |
|--------|--------|
| **صورة + تفاصيل** | اسم، سعر (مع شطب إن وجد خصم)، كمية المخزون |
| **الوصف** | نص الوصف مع "See More" إن طال |
| **خيارات المنتج (Variants)** | إذا كان للمنتج `choice_options` (مثلاً لون، حجم): اختيار من أزرار راديو |
| **الكمية** | حقل عدد مع أزرار +/- وحد أقصى = المخزون |
| **السعر الديناميكي** | عند تغيير الخيارات أو الكمية يُحدَّث السعر والمخزون عبر AJAX |
| **إضافة/تحديث السلة** | زر "Add to Cart" أو "Update Cart" حسب وجود المنتج في السلة |

### 1.3 قسم الفوترة (يمين الصفحة)
| الميزة | الوصف |
|--------|--------|
| **اختيار العميل** | Select2 بقائمة العملاء (من جدول users) + خيار "Walk In Customer" (مسح الاختيار) |
| **تفاصيل العميل** | عند اختيار عميل: الاسم، الهاتف، الإيميل في صندوق تحت الـ select |
| **إضافة عميل جديد** | زر "+ Add New Customer" داخل قائمة Select2 يفتح modal (الاسم، الإيميل، الهاتف) ثم حفظ عبر `customer-store` |
| **جدول السلة** | عناصر السلة: صورة، اسم، متغير، كمية (قابلة للتعديل)، السعر الفرعي، زر حذف |
| **حد أقصى للكمية** | حقل الكمية له `max` حسب المخزون؛ عند تجاوزه يُعرض تنبيه ويُحدَّث إلى الحد الأقصى |

### 1.4 الخصومات والضرائب
| الميزة | الوصف |
|--------|--------|
| **خصم المنتج** | يُحسب تلقائياً من إعدادات المنتج ويُعرض تحت "Product discount" |
| **كوبون** | زر بجانب "Coupon Discount" يفتح modal: قائمة كوبونات متاحة للعميل + حقل إدخال كود. **متاح فقط لعميل مسجّل** (معطل لـ Walk In) |
| **خصم إضافي** | modal "update_discount": مبلغ أو نسبة مع حد أقصى (المبلغ الكلي أو 100%) |
| **نقاط الولاء** | زر بجانب "Loyalty points discount": استخدام نقاط العميل. **يتطلب عميل مسجّل + تفعيل برنامج الولاء** |
| **الضريبة** | تُحسب من إعدادات المنتج وتُعرض في "tax" |

### 1.5 الدفع وإتمام الطلب
| الميزة | الوصف |
|--------|--------|
| **طريقة الدفع** | نقد (Cash) أو بطاقة (Card) — اختيار من زرّين |
| **المبلغ المدفوع** | حقل "paid_amount" (للمبلغ النقدي قيمته الافتراضية = الإجمالي؛ للبطاقة readonly = الإجمالي) |
| **المتبقي/الباقي** | حقل "change_amount" يُحدَّث ديناميكياً عند تغيير المبلغ المدفوع |
| **Place Order** | زر إتمام الطلب؛ يُعطّل إذا كان المدفوع أقل من الإجمالي (مع tooltip توضيحي) |
| **Cancel Order** | إفراغ السلة وإعادة تحميل الصفحة |

### 1.6 بعد إتمام الطلب
| الميزة | الوصف |
|--------|--------|
| **طباعة الفاتورة** | بعد الطلب يظهر modal "Print Invoice" مع زر "Proceed, If thermal printer is ready" لطباعة منطقة الفاتورة |
| **محتوى الفاتورة** | اسم المتجر، الفرع، العنوان، الهاتف، رقم الطلب، التاريخ، بيانات العميل، جدول (كمية، وصف، سعر)، الخصومات، الضريبة، الإجمالي، طريقة الدفع |

### 1.7 قائمة طلبات POS
| الميزة | الوصف |
|--------|--------|
| **الصفحة** | `GET /admin/pos/orders` — قائمة طلبات من نوع `pos` مع فلتر تاريخ (start_date, end_date) وبحث (رقم الطلب، الحالة، المرجع) |
| **تصدير** | تصدير الطلبات حسب الفلتر الحالي |
| **تفاصيل الطلب** | رابط لصفحة تفاصيل الطلب (نفس عرض الطلب العادي) |
| **طباعة فاتورة** | زر طباعة الفاتورة من قائمة الطلبات |

## 2. الفجوات والمشاكل الحالية

### 2.1 واجهة المستخدم وسهولة الاستخدام
- **لا يوجد رابط سريع من صفحة POS إلى طلبات POS:** الشريط الجانبي يحتوي فقط على "POS" → الصفحة الرئيسية. للوصول لطلبات POS يجب معرفة الرابط أو البحث في القوائم إن وُجد.
- **زر "Add to cart" بالإنجليزية فقط** في `_single_product.blade.php` دون استخدام `translate()`.
- **عدم وضوح حالة "Walk In Customer":** عند عدم اختيار عميل، الكوبون ونقاط الولاء معطّلان دون رسالة واضحة في نفس الصفحة (يوجد tooltip على الأيقونة فقط).
- **لا اختصار لوحة مفاتيح** للبحث أو لإتمام الطلب (مثلاً Enter في حقل البحث، أو Ctrl+Enter لـ Place Order).

### 2.2 المنتجات والسلة
- **المنتجات ذات المخزون 0 لا تظهر** في قائمة المنتجات (`total_stock > 0`) — منطقي للبيع، لكن قد يُراد إظهارها مع تعطيل الإضافة أو تنبيه "نفد".
- **لا دعم مسح ضوئي للباركود:** لا حقل بحث بباركود ولا ربط منتج بـ barcode/سريال.
- **تحديث الكمية في السلة:** يتم عبر حقل رقمي فقط؛ لا أزرار +/− سريعة بجانب كل صف في السلة (الكمية موجودة لكن التعديل من لوحة المفاتيح فقط في بعض الواجهات).

### 2.3 الدفع والفاتورة
- **طريقتا دفع فقط:** نقد وبطاقة. لا خيار "محفظة" (Wallet) من واجهة POS الحالية رغم وجود نظام محفظة في التطبيق.
- **لا إيصال إلكتروني:** لا إرسال فاتورة بالبريد أو تحميل PDF من نفس الشاشة بعد الطلب.
- **طباعة الفاتورة:** تعتمد على `window.print()` فقط؛ لا تنسيق مخصص للطابعة الحرارية (عرض ثابت، خطوط مناسبة).

### 2.4 العملاء والكوبونات
- **قائمة العملاء محمّلة كلها في الـ select** (من `$users`) وليست بحثاً آلياً (AJAX) في الصفحة الأولى؛ قد يكون بطيئاً مع عدد كبير من العملاء. (يوجد route `getCustomers` لـ AJAX لكن الـ view تستخدم قائمة ثابتة في index.)
- **الكوبونات:** تُعرض قائمة للعميل المحدد؛ لا تحقق فوري عند إدخال الكود قبل الضغط على Apply (لا تحقق لحظي من صلاحية الكود).

### 2.5 التقني
- **تعامل مع المنتجات بدون متغيرات:** إذا كان `choice_options` فارغاً أو null قد يظهر خطأ في `quickViewModalFooter` أو `variantPrice` (مراجعة `json_decode($product->choice_options)`).
- **الجلسة:** السلة و`customer_id` و`branch_id` في Session؛ عند فتح أكثر من تاب قد يتشاركان نفس السلة أو نفس العميل ما لم يُصمّم بشكل صريح لتابات متعددة.

## 3. التحسينات المقترحة (مرتبة حسب الأثر وسهولة التنفيذ)

### 3.1 تحسينات سريعة — سهولة الاستخدام
1. **رابط "طلبات نقطة البيع" من صفحة POS:** إضافة زر أو رابط في الهيدر أو أعلى قسم الفوترة: "عرض طلبات اليوم" أو "سجل الطلبات" → `route('admin.pos.orders')`.
2. **ترجمة زر "Add to cart":** استخدام `{{ translate('Add to cart') }}` في `_single_product.blade.php`.
3. **تنبيه واضح للعميل العابر:** رسالة صغيرة تحت اختيار العميل: "عميل عابر — الكوبون ونقاط الولاء غير متاحة" عند عدم اختيار عميل.
4. **اختصار لوحة المفاتيح:**  
   - Enter في حقل البحث → تنفيذ البحث.  
   - (اختياري) Ctrl+Enter أو زر واضح "إتمام الطلب" يبقى في مجال الرؤية عند التمرير.

### 3.2 تحسينات متوسطة
5. **أزرار +/− للكمية في السلة:** بجانب حقل الكمية في كل صف لزيادة/تقليل سريعة دون الكتابة.
6. **دعم الباركود (إن وُجد حقل في المنتجات):** حقل بحث "باركود" أو تركيز تلقائي على حقل عند فتح الصفحة؛ عند المسح يضاف المنتج مباشرة بكمية 1 (أو يزيد الكمية إن كان في السلة).
7. **تحقق فوري من الكوبون:** عند إدخال الكود (مثلاً بعد فقدان التركيز أو زر "تحقق") عرض صلاحية الكود قبل الضغط على Apply.
8. **تنسيق طباعة للطابعة الحرارية:** CSS مخصص للطباعة (عرض 80mm/58mm، خط بسيط، بدون ألوان ثقيلة) وزر "طباعة حرارية" إن لزم.

### 3.3 تحسينات لاحقة
9. **دفع بالمحفظة:** إن كان العميل مسجّل ولديه رصيد، خيار "دفع من المحفظة" أو "جزء من المحفظة" مع تحديث المبلغ المطلوب.
10. **إيصال إلكتروني:** بعد الطلب: زر "إرسال الفاتورة بالبريد" أو "تحميل PDF".
11. **عرض المنتجات منخفضة المخزون في POS:** علامة على البطاقة (مثلاً شارة "مخزون منخفض") دون منع الإضافة؛ أو فلتر اختياري "إظهار نفاد المخزون" مع تعطيل الإضافة لها.
12. **دعم تعدد الفروع في الـ Session:** إذا كان النظام متعدد الفروع، التأكد من اختيار الفرع عند الدخول لـ POS وربط الطلب والمنتجات بالفرع الصحيح.

## 5. الخلاصة

صفحة POS الحالية تغطي: **عرض منتجات مع تصنيف وبحث، معاينة سريعة مع متغيرات، سلة مع خصم وكوبون ونقاط ولاء، اختيار عميل أو عميل عابر، دفع نقدي/بطاقة، وطباعة فاتورة بعد الطلب وقائمة طلبات POS.**

أهم التحسينات من ناحية **سهولة الاستخدام** و**إضافات بسيطة**: ربط الصفحة بقائمة طلبات POS، ترجمة النصوص الثابتة، توضيح حالة العميل العابر، وأزرار +/− للكمية في السلة. ثم يأتي دعم الباركود والتحقق الفوري من الكوبون وتنسيق الطباعة الحرارية إذا كانت مطلوبة في بيئة المتجر.

تم إعداد هذا الملف كمرجع لتطوير وتحسين صفحة نقطة البيع.
