# تعليمات الرفع وتحديث قاعدة البيانات

استخدم الملف **upload-full-with-database.zip** لرفع كافة التعديلات مع تطبيق تحديثات قاعدة البيانات على السيرفر.

---

## 1) نسخ احتياطي لقاعدة البيانات (مهم جداً)

قبل أي شيء، اعمل نسخة احتياطية من قاعدة البيانات على السيرفر (من cPanel أو phpMyAdmin أو أوامر mysqldump).

---

## 2) رفع الملفات

1. ارفع **upload-full-with-database.zip** إلى **جذر المشروع** على الاستضافة (نفس مستوى المجلدات `app` و `resources` و `database`).
2. **فك الضغط (Extract)** داخل مجلد المشروع حتى تُستبدل الملفات وتُحدَّث مجلد `database/migrations` في مكانه.

---

## 3) تحديث قاعدة البيانات على السيرفر

من التيرمنال (SSH أو Terminal في cPanel):

```bash
cd مسار_المشروع
php artisan migrate
```

- إذا ظهر سؤال **"Do you want to run the migrations?"** اكتب **yes** ثم Enter.
- لن تُنفَّذ إلا الـ migrations التي لم تُنفَّذ سابقاً؛ الباقي سيُتخطى تلقائياً.

---

## 4) مسح الكاش والأوامر الإضافية

```bash
php artisan view:clear
php artisan config:clear
php artisan cache:clear
php artisan route:clear
php artisan optimize
```

---

## 5) التأكد على السيرفر

- وجود الملف: **`public/assets/admin/js/jquery.js`**
- وجود الملف: **`public/assets/admin/js/bootstrap.js`**
- ملف **`.env`** يحتوي إعدادات قاعدة البيانات الصحيحة للسيرفر.
- (اختياري) إظهار قائمة نقطة البيع: أضف في `.env` السطر: **`APP_SHOW_POS_MENU=true`**

---

## 6) في حال حدوث خطأ أثناء migrate

- راجع رسالة الخطأ.
- تأكد أن بيانات الاتصال بقاعدة البيانات في `.env` صحيحة.
- إذا كان الخطأ يتعلق بجدول أو عمود موجود مسبقاً، قد تحتاج لتعديل الـ migration أو تشغيل migrate يدوياً حسب الحالة.

---

*بعد تنفيذ الخطوات أعلاه تكون كافة الملفات مرفوعة وقاعدة البيانات محدَّثة.*
