# خطة تنفيذ ميزة "إشعارات عند نفاذ الكمية" (Stock Alerts)

**الهدف:** تنبيه المسؤول عند نفاذ كمية المنتج أو وصول المخزون إلى حد أدنى، مع إمكانية (اختيارية) إشعار العملاء عند عودة المنتج للمتجر.

**ملاحظة:** لا يُنفذ أي كود حتى تقول "ابدأ".

---

## تحليل النظام الحالي

### ما هو موجود بالفعل

| المكون | الوصف |
|--------|--------|
| **جدول المنتجات** | عمود `total_stock` (المخزون الحالي)، وعمود `minimum_stock_alert` (حد التنبيه لكل منتج، اختياري). |
| **إعداد عام** | في `business_settings`: مفتاح `default_minimum_stock_alert` (قيمة افتراضية 5 عند عدم تحديد حد لكل منتج). |
| **منطق "مخزون منخفض"** | في `App\Models\Product`: scope `lowStock($defaultAlert)` يحدد المنتجات التي `total_stock <= minimum_stock_alert` أو (إن كان minimum_stock_alert فارغاً) `total_stock <= default`. |
| **لوحة التحكم** | في Dashboard يتم جلب `low_stock_products` (أول 10 منتجات) وعرضها في partial `_low-stock-products.blade.php`. |
| **قائمة المنتجات** | فلتر `stock_filter=low_stock` يعرض المنتجات ذات المخزون المنخفض فقط. |
| **تحديث المخزون** | يتم خصم الكمية من `total_stock` (وربما من `variations`) عند تأكيد الطلب في: API OrderController، Admin OrderController، Branch OrderController، POS. |

### ما لا يوجد بعد

- **إشعار فوري (صوت + نافذة منبثقة)** للمسؤول عند وجود منتجات مخزونها منخفض أو نفد، أثناء تصفحه لوحة التحكم (مثل إشعار "طلب جديد" أو "رسالة تواصل معنا").
- آلية **"تجاهل الآن"** أو **"لا تُظهر مرة أخرى حتى يرتفع المخزون"** (اختياري).
- **إشعار للعميل** عند عودة منتج كان نفد (يتطلب تسجيل رغبة "أخبرني عند التوفر" أو ربطه بقائمة الأمنيات).

## المرحلة 1: إشعار المسؤول (صوت + Popup)

### 1.1 إرجاع عدد منتجات المخزون المنخفض في الـ Polling

- **الملف:** `App\Http\Controllers\Admin\SystemController`
- **الدالة:** `restaurantData()`
- **الإضافة:** حساب عدد المنتجات ذات المخزون المنخفض باستخدام نفس منطق الـ Dashboard:
  - `$defaultStockAlert = (int) (Helpers::get_business_settings('default_minimum_stock_alert') ?? 5);`
  - `$lowStockCount = Product::lowStock($defaultStockAlert)->count();`
- إضافة إلى مصفوفة `data` المُرجعة: **`low_stock_count`** = `$lowStockCount`.

### 1.2 واجهة لوحة التحكم (صوت + نافذة منبثقة)

- **الملف:** `resources/views/layouts/admin/app.blade.php`
- **الإضافات:**
  - **Modal جديد:** مثلاً `id="popup-modal-low-stock"`، بنفس أسلوب `popup-modal-contact` و`popup-modal`:
    - عنوان مثل: "يوجد منتجات مخزونها منخفض أو نفد. تحقق من قائمة المنتجات."
    - زر "تجاهل الآن" (إغلاق النافذة فقط، أو استدعاء route لتسجيل "تجاهل" إن رُغب لاحقاً).
    - زر "عرض المنتجات" يوجّه إلى: `route('admin.product.list', ['stock_filter' => 'low_stock'])`.
  - في السكربت الذي يستدعي `get-restaurant-data` كل 10 ثوانٍ:
    - بعد التحقق من `new_order` و`pending_type_approval` و`new_contact_us`، إضافة شرط: إذا **`data.low_stock_count > 0`**:
      - استدعاء **`playAudio()`** (نفس صوت الإشعارات الحالي).
      - إظهار **`#popup-modal-low-stock`**.

ترتيب الأولوية في العرض (مقترح): طلب جديد → موافقة نوع مستخدم → رسالة تواصل → **مخزون منخفض**.

### 1.3 (اختياري) تخفيف الإزعاج: "تجاهل مؤقت"

- إن رُغب بعدم إظهار إشعار المخزون لمدة محددة بعد النقر "تجاهل الآن":
  - حفظ في **Session** مثلاً: `session(['low_stock_alert_dismissed_at' => now()])` عبر route مثل `GET /admin/ignore-check-low-stock`.
  - في `restaurantData()`: إذا وُجد `session('low_stock_alert_dismissed_at')` وكانت المدة منذ آخر تجاهل أقل من (مثلاً) 30 دقيقة، لا تُضاف `low_stock_count` إلى الاستجابة أو تُعاد بقيمة 0.
  - يمكن تنفيذ هذا لاحقاً دون تعقيد الخطة الأساسية.

### 1.4 الترجمات

- إضافة مفاتيح في `resources/lang/ar/messages.php` و`en/messages.php`:
  - عنوان الـ modal، نص الأزرار، أي توضيح (مثل: "عرض المنتجات ذات المخزون المنخفض").

## المرحلة 2: (اختياري) إشعار العملاء عند عودة المنتج

هذه المرحلة **خارج النطاق الأول** ويمكن تنفيذها لاحقاً.

### 2.1 الفكرة

- عندما يصبح منتج **خارج المخزون** (`total_stock <= 0`) يريد بعض العملاء أن يُشعَروا عند **عودة الكمية** (`total_stock > 0`).
- يتطلب ذلك:
  - **جدول** لتسجيل "رغبة الإشعار" (مثلاً: `product_stock_notify` أو استخدام علاقة مع قائمة الأمنيات إن وُجدت): `user_id` أو `guest_id`، `product_id`، `notified_at` (nullable)، `created_at`.
  - في الواجهة الأمامية/التطبيق: زر "أخبرني عند التوفر" يرسل طلباً (API أو Web) لتسجيل السجل.
  - عند **تحديث المخزون** (إضافة كمية لمنتج كان total_stock = 0): استعلام عن الطلبات غير المُشعَر بها، إرسال إشعار (بريد أو push) ثم تحديث `notified_at`.

### 2.2 ملاحظة

- لا يُنفذ في الخطة الحالية؛ يُترك كمرحلة لاحقة أو ميزة منفصلة.

## ترتيب التنفيذ المقترح (للمرحلة 1 فقط)

1. **تعديل `SystemController::restaurantData()`:** إضافة `low_stock_count` باستخدام `Product::lowStock($defaultStockAlert)->count()`.
2. **تعديل `layouts/admin/app.blade.php`:** إضافة modal "مخزون منخفض" + في سكربت الـ polling شرط `low_stock_count > 0` مع تشغيل الصوت وإظهار الـ modal وربط الأزرار.
3. **الترجمات:** إضافة النصوص العربية والإنجليزية للـ modal والأزرار.
4. **(اختياري لاحقاً)** Route وController لتجاهل إشعار المخزون مؤقتاً مع Session.

## خطوات اختبار مقترحة (بعد التنفيذ)

1. التأكد من وجود منتج بمخزون أقل من حد التنبيه (أو استخدام الافتراضي 5).
2. فتح لوحة التحكم (أي صفحة admin) وانتظار مدة لا تزيد عن 10 ثوانٍ.
3. التحقق من تشغيل الصوت وظهور نافذة "منتجات مخزون منخفض".
4. النقر "عرض المنتجات" والتأكد من التوجيه إلى قائمة المنتجات مع فلتر `stock_filter=low_stock`.
5. رفع مخزون المنتجات المعنية فوق الحد والتأكد من عدم ظهور الإشعار مرة أخرى (أو حسب منطق "تجاهل" إن وُجد).
