# خطة تحسين واجهة اختيار التصنيف الأب (Parent Select UX)

> **المشكلة:** قائمة اختيار "التصنيف الرئيسي" غير واضحة للمستخدم — الترميز الحالي (`— — اسم`) لا يوضح الهرمية بشكل كافٍ.

---

## 1. المشكلة الحالية

### 1.1 ما يظهر للمستخدم

```
<select name="parent_id">
  <option>إكسسوارات</option>
  <option>سجائر إلكترونية</option>
  <option>— سوائل نيكوتين</option>
  <option>— — سوائل نيكوتين ملح</option>
</select>
```

### 1.2 نقاط الغموض

| النقطة | السبب |
|--------|-------|
| معنى `—` | المستخدم قد لا يفهم أن الشرطة = مستوى فرعي |
| ترتيب الخيارات | الجذور والفرعيات مخلوطة — صعب التمييز |
| نص التسمية | "التصنيف الرئيسي" قد يُفهم خطأ — المستخدم يختار "تحت ماذا" وليس "رئيسي" |
| عدم وضوح المسار | لا يظهر المسار الكامل (أب › ابن › ابن ابن) |

---

## 2. الحلول المقترحة

### 2.1 الخيار أ: عرض المسار الكامل (Breadcrumb) — **موصى به**

**الفكرة:** بدل `— — سوائل نيكوتين ملح` نعرض المسار من الجذر حتى التصنيف:

```
سجائر إلكترونية › سوائل نيكوتين › سوائل نيكوتين ملح
```

**التنفيذ:**
- في `buildParentCategoryOptions`: بناء `path` لكل تصنيف (من الجذر حتى الأب المباشر).
- عرض: `اسم التصنيف (تحت: المسار)` أو `المسار › اسم التصنيف` للفرعيات.

**مثال النتيجة:**
```
إكسسوارات
سجائر إلكترونية
سوائل نيكوتين (تحت: سجائر إلكترونية)
سوائل نيكوتين ملح (تحت: سجائر إلكترونية › سوائل نيكوتين)
```

**مميزات:** واضح، لا يحتاج واجهة جديدة.  
**عيوب:** النص قد يطول للتصنيفات العميقة.

---

### 2.2 الخيار ب: قوائم متتالية (Cascading Selects)

**الفكرة:** بدل قائمة واحدة، عدة قوائم:
1. **التصنيف الرئيسي** (جذور فقط)
2. **التصنيف الفرعي** (يظهر بعد اختيار 1 — أبناء التصنيف المختار)
3. **التصنيف الفرعي الفرعي** (يظهر إن وُجد — أبناء 2)

المستخدم يختار مستوى مستوى. القيمة النهائية = آخر قائمة يختار منها.

**التنفيذ:**
- استبدال `<select name="parent_id">` بـ 2–3 selects مع `data-parent` أو `id` للربط.
- JavaScript يملأ القائمة التالية عند تغيير السابقة.
- إرسال `parent_id` = قيمة آخر select مُختار.

**مميزات:** بسيط للمستخدم، كل قائمة قصيرة.  
**عيوب:** يحتاج JS، وتعديل في الـ Controller لتمرير البيانات.

---

### 2.3 الخيار ج: تحسين الترميز الحالي

**الفكرة:** الإبقاء على قائمة واحدة مع تحسين العرض:

| التحسين | الوصف |
|---------|-------|
| أيقونة/رمز | استبدال `—` برمز مثل `└` أو `↳` أو سهم |
| مسافة بادئة مرئية | استخدام `padding-inline-start` أو `&nbsp;` حسب العمق |
| تجميع بصري | `<optgroup>` للتجميع (محدود في HTML) |
| تسمية أوضح | "ضع التصنيف الجديد تحت:" بدل "التصنيف الرئيسي" |

**مثال:**
```
↳ إكسسوارات
↳ سجائر إلكترونية
  ↳ سوائل نيكوتين
    ↳ سوائل نيكوتين ملح
```

**ملاحظة:** `<option>` لا يدعم تنسيق داخلي جيد — المسافات والرموز قد تظهر متشابهة في بعض المتصفحات.

---

### 2.4 الخيار د: شجرة قابلة للنقر (Tree Picker) — **تم تنفيذه**

**الفكرة:** بدل select، عرض شجرة قابلة للتوسيع/الطي. المستخدم ينقر على التصنيف الذي يريده كأب.

**ما تم تنفيذه:**
- شجرة HTML/CSS/JS بدون مكتبات خارجية.
- توسيع/طي عند النقر على السهم.
- اختيار عند النقر على الصف — يتم تعيين `parent_id` في حقل مخفي.

---

## 3. التوصية والترتيب

| الأولوية | الخيار | الجهد | الوضوح |
|----------|--------|-------|--------|
| 1 | **أ — المسار الكامل** | منخفض | عالي |
| 2 | **ج — تحسين الترميز + تسمية** | منخفض | متوسط |
| 3 | **ب — قوائم متتالية** | متوسط | عالي |
| 4 | **د — شجرة** | عالي | عالي جداً |

**الخطوة الأولى المقترحة:** تنفيذ **الخيار أ** (عرض المسار الكامل) مع تحسين التسمية:

- تسمية الحقل: **"وضع التصنيف الجديد تحت:"** أو **"التصنيف الأب:"**
- نص مساعد: "اختر التصنيف الذي سيكون التصنيف الجديد فرعاً منه"

---

## 4. خريطة التعديلات (للخيار أ)

| الملف | التعديل |
|-------|---------|
| `CategoryController::buildParentCategoryOptions` | إضافة حقل `path` أو `full_label` يعرض المسار من الجذر |
| `sub-index.blade.php` | استخدام `full_label` أو `path` في عرض الخيارات |
| `edit.blade.php` | نفس التعديل |
| ملفات الترجمة | إضافة مفتاح مثل `category_parent_select_label` |

---

## 5. مثال التسميات بعد التعديل

| العمق | التسمية الحالية | التسمية المقترحة |
|-------|-----------------|-------------------|
| 0 | إكسسوارات | إكسسوارات |
| 0 | سجائر إلكترونية | سجائر إلكترونية |
| 1 | — سوائل نيكوتين | سوائل نيكوتين ← تحت: سجائر إلكترونية |
| 2 | — — سوائل نيكوتين ملح | سوائل نيكوتين ملح ← تحت: سجائر إلكترونية › سوائل نيكوتين |

---

## 6. ملخص

1. **المشكلة:** قائمة الأب غير واضحة — الشرطات لا توضّح الهرمية.
2. **الحل الموصى به:** عرض المسار الكامل (Breadcrumb) + تحسين تسمية الحقل.
3. **البديل:** قوائم متتالية (Cascading) إن رغب المستخدم بواجهة أبسط.
